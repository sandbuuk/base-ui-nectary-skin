## Use history.pushState for links

We enhanced our markdown to support links that uses history.pushState instead of an anchor tag. To do this add the `{use-history}` option after the basic markdown link definition:

## Use External links

We enhanced our markdown to support [external links](http://localhost:5000/?version=latest&path=/components/link/examples#external). To do this add the `{external}` option after the basic markdown link definition:

<Example Component={ExternalLinkExample}/>

## Clickable links

We usually prefer to reserve links for actual navigation and use buttons for actions, sometimes you need to make a link clickable without navigating, e.g. to open up a modal.

Please note that **it is not a recommended ux pattern**.

To do this add the `preventDefault` attribute along with some id, after the basic markdown link definition of the link you want to make clickable and then listen for the click event on the `sinch-rich-text` component:

<Example Component={ClickableLinkExample}/>
