# Examples

Cards are containers that group related information. They help to organize and divide content making it easier to scan big chunks of information.

## Simple

A simple card in Nectary is a user interface element that is used to present information or content in a compact, organized manner. It typically consists of a container with a fixed or variable size, and includes only a title and text. They can be used in a variety of contexts, such as listing products or services, showcasing articles or blog posts, or displaying information about a particular topic.

The title and text of the simple card can be customized to fit the needs of the design and the context in which it is being used.

## Icon

The icon can be used to add visual interest or context to the card. The design and layout of the card is kept minimal and straightforward, with a focus on presenting information in a clear and concise manner.

<Example Component={IconExample} />

## Complex

The complex card combines multiple card features into a single component. It includes an illustration media section, a title with an icon, body content, and a footer with an action button. This type of card is ideal for rich content presentations where you need to display multiple pieces of related information and enable user interaction.

<Example Component={ComplexExample} />

## Clickable

The clickable card turns the entire card into an interactive element that responds to click events. This variation is useful when you want the whole card to be clickable rather than just specific elements within it. It's commonly used in grids or lists where each card represents a selectable item or navigates to more detailed content.

<Example Component={ClickableExample} />

## Selected

The selected card visually indicates an active or chosen state. This variation is useful in interfaces where users can select one or multiple cards from a collection, such as in selection workflows or configuration interfaces. The selected state provides clear visual feedback about which items have been chosen.

<Example Component={SelectedExample} />

## Disabled

The disabled card represents an inactive or unavailable state. This variation is used when you need to show content that exists but is not currently accessible or interactive. The disabled state helps communicate to users that the card's functionality is temporarily or permanently unavailable while still maintaining the information's visibility.
