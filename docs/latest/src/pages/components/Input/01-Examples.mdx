## Mask

Masked Input applies only for fixed length inputs known in advance. For example Bank Card number, or Phone number.

We have decided to use monospace font for masked inputs to avoid unnecessary jumping of placeholder symbols.

⚠️ Make sure you have enough width to fit the whole mask.

See [Phone Input](?path=/compositions/phoneinput/examples), [Date Input](?path=/compositions/dateinput/examples) or [Time Input](?path=/compositions/timeinput/examples) for more examples.

### Syntax

In mask syntax any letter is represented by `A` symbol:

```tsx
<sinch-input mask="AAAA"></sinch-input>
```

Such mask will allow to input only four any letters, like `abcd`.

Any digit is represented by `0` symbol:

```tsx
<sinch-input mask="0000"></sinch-input>
```

Such mask will allow to input only four any digits, like `1234`.

You can use any other symbols to represent non-editable mask placeholders:

```tsx
<sinch-input mask="+46-000"></sinch-input>
```

Such mask will have non-editable `+46-` part, followed by 3 editable digits.

In case you need to use `A` or `0` as non-editable placeholder part then it needs to be escaped with `\` symbol:

```tsx
<sinch-input mask="+46-\000"></sinch-input>
```

Such mask will have non-editable `+46-0` part, followed by 2 editable digits.

### Placeholder

By default any editable letters or digits are visually presented by `_` symbols. But it is possible to customize this:

```tsx
<sinch-input mask="00-00-0000@@dd/mm/yyyy"></sinch-input>
```

Here `@@` is a delimiter between mask value and placeholder representation. For example if user types `01012022`, theactual input value will be `01-01-2022`, but visual representation will be `01/01/2022`.
