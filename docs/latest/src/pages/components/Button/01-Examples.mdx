# Examples

The button is a clickable element that trigger events or actions.

## Type

### Primary

A primary button is a type of user interface element that is used to initiate the most important action on a page or screen. It is typically represented as a labelled button that stands out visually from other buttons or controls on the page, and is often used to confirm a user's intention or to submit a form.

<Example Component={TypePrimaryExample}/>

### Secondary

A secondary button is always used in combination with a primary button. It must be associated with a less important action compared to the primary button. The label on a secondary button will typically be less urgent or less compelling than the primary button, such as "Advanced" or "Cancel".

<Example Component={TypeSecondaryExample}/>

### CTA Primary

A call to action (CTA) button is a type of user interface element that is used to encourage the user to take a specific action.  It is typically designed to be attention-grabbing and easy to click or tap on. The label on a CTA button will typically be a strong call to action, such as "Sign Up", "Purchase”, “Subscribe”, “Upgrade”

<Example Component={TypeCtaPrimaryExample}/>

### CTA Secondary

The secondary CTA button is always used in combination with a primary CTA button. It must be associated with a less important action compared to the primary button. The label on a secondary button will typically be less urgent or less compelling than the primary button, such as "Advanced" or "Cancel".

<Example Component={TypeCtaSecondaryExample}/>

### Destructive

A destructive button is a type of user interface element that is used to warn the user about a potentially dangerous or destructive action that cannot be reverted.
The label on a danger button will typically be a strong warning, such as "Delete" or "Cancel", to emphasize the potentially destructive nature of the action

<Example Component={TypeDestructiveExample}/>

## Disabled

Disabled buttons are often used to indicate to the user that a certain action is not currently available or cannot be performed. For example, a disabled button might be used to indicate that a form cannot be submitted because required fields have not been filled in, or that a certain action cannot be performed because the user does not have the necessary permissions.The label on a disabled button it is always the same as an enabled button, but it will be unclickable or otherwise not functional.

<Example Component={DisabledExample}/>

## Spinner

A button with a loading state is a user interface element that is used to indicate to the user that an action is currently being performed. It is represented as a labelled button that includes  a spinning animation to the left of the label. When a button is in a loading state are always disabled, to prevent the user to trigger a action multiple times while the action is still loading.

<Example Component={SpinnerExample}/>

## Left Icon

By including a left icon in a button, the button's purpose can be communicated more quickly and clearly to the user. For example, a button with a trash can icon is likely to be understood as a button for deleting something, while a button with a pencil icon is likely to be understood as a button for editing something
