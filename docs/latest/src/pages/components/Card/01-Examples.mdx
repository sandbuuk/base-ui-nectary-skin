# Examples

Cards are versatile containers that group related information into organized, scannable sections. The card has 4 available slots that appears in the following order:

1. `media`
2. `title`
3. `content`
4. `footer`

## Simple Card

A simple card using only the `title` and `content` slots.

<Example Component={SimpleExample} />

## Complex

A complete card utilizing all four available slots, including the `media` slot for illustrations or other media-related content.

<Example Component={ComplexExample} />

## Title Variants

The `sinch-card-v2-title` component supports an optional icon slot to display icons alongside titles. The icon placement can be controlled with the `orientation` prop:

* `orientation="horizontal"` (default): Icon appears beside the title
* `orientation="vertical"`: Icon appears above the title

<Example Component={TitlesExample} />

## Clickable

A clickable card provides visual feedback through `:hover` and `:active` state styles, but does not include any built-in interactive functionality. There are two ways to apply this visual style:

* Automatically: When a click event handler is added to the card (`-click` or `on-click` depending on the framework)
* Explicitly: By adding the `clickable` attribute
