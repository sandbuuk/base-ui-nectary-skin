# Examples

Popover is a low-level building block that we use to implement other components such as Dialog, Dropdown, Action Menu and Select.

It's stateless, as well as all Nectary components, and therefore it's up to a consumer to decide when and how to open/close it.

## Non-modal

Our non-modal popover is designed to provide useful information without taking focus away from the main content. It's like a helpful tooltip or a smart autocomplete feature for text fields.

Add `allowScroll` prop to allow scrolling when popover is open.

<Example Component={NonModalExample}/>

## Modal

Our modal popover captures the user's attention and *requires* them to interact with it. It functions like a dialog box and is an effective way to present important information or options to the user.

<Example Component={ModalExample}/>

## Orientation

Orientation in a popover component refers to the position of the popover relative to the element that triggers it. Popovers can be oriented in a variety of ways, including top, bottom, left, and right.

The orientation of a popover can help to ensure that the popover is visible to the user and does not overlap with other content on the page. It can also be used to provide a more intuitive or aesthetically pleasing layout for the popover and its trigger element.

See API section for all the possible values.

Orientation `top` and `bottom` will stretch the popover body to the target's width, like in Select example.
