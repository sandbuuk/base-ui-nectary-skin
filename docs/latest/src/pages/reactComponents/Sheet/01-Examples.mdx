# Examples

A sheet component is a slide-in drawer that allows users to view and interact with content in a panel that slides from the edge of the screen. It is typically used to display additional information, navigation options, or contextual actions without leaving the current page.

The sheet component consists of a container that slides in from one of the four edges (left, right, top, or bottom) and overlays or pushes the main content. It includes a title, scrollable content area, and optional action buttons. The sheet can be dismissed by the user by clicking a close button, clicking outside the sheet (for modal overlays), or pressing the ESC key.

Sheets are ideal for displaying detailed information, forms, settings panels, or navigation menus in a way that maintains context with the main application. They can be configured with different placement options and overlay modes to suit various use cases.

## Push Overlay

**Multiple sheets are not supported.**

The sheet can be configured with `overlay="push"` mode, which pushes the main content aside instead of overlaying it. This example demonstrates how to listen to animation events and apply transitions to your page layout.

There are two ways to animate your content when using the push overlay mode:
1. **Using Animation Events**: The sheet provides `onSheetAnimationStart` callback when the push animation starts. You can use this callback to trigger animations on your content. Check the example below for implementation details.
2. **Using CSS Variables**: The sheet exports its dimensions via CSS variables (`--sinch-sheet-export-current-width` and `--sinch-sheet-export-current-height`) that can be used to animate your content.

<Example Component={PushOverlayExample}/>

### CSS Variables

When using `overlay="push"`, the sheet exports its current dimensions as CSS variables on `document.body`:

* `--sinch-sheet-export-current-width`: Current width of the sheet (e.g., `400px`)
* `--sinch-sheet-export-current-height`: Current height of the sheet (e.g., `500px`)

These variables are available during animations and can be used to transition your page layout. They are set when the sheet opens and removed when it closes.

## Animation

The `onSheetAnimationEnd` callback is useful to know when the sheet has finished its opening or closing animation. This can be particularly helpful for triggering actions that should only occur after the sheet is fully visible or hidden, such as navigating to a different page, loading content, or updating the UI.

## Accessibility

The sheet component should have an `aria-label` attribute for screen reader accessibility. This label should describe the purpose of the sheet to users who cannot see it.

```tsx
<Sheet
  open={isOpen}
  onClose={() => setIsOpen(false)}
  aria-label="Settings panel"
  title={<SheetTitle title="Settings" onClose={() => setIsOpen(false)}/>}
>
  {/* content */}
</Sheet>
```

## Close Event Details

The `onClose` callback receives a detail string indicating how the sheet was closed:
- `'close'` - Close button was clicked
- `'escape'` - Escape key was pressed
- `'backdrop'` - User clicked outside the sheet

```tsx
<Sheet
  open={isOpen}
  onClose={(detail) => {
    if (detail === 'escape') {
      // User pressed Escape
    }
    setIsOpen(false)
  }}
/>
```
