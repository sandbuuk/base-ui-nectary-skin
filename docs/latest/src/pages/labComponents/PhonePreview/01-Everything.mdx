## Custom

**Note: in order to keep style aligned across different applications that use this component, it is preferred to keep these values default.**

The phone container custom element has an inline size of 288px but shrinks if the parent container is smaller than that. If you want to have a larger base inline size, it can be set as an inline style, e.g.:

```css
inline-size: min(288px, 100%); /* this is the default */

inline-size: min(30rem, 100%);
inline-size: 100%;
```

The content scales and shrinks proportional to the base inline size. The default base inline is also 288px,and can be modified to affect the scaling behavior of the content using a custom property.

If modified, it **MUST BE SET IN PX**.

```css
--base-size: 288px; /* this is the default */

--base-size: 350px; /* ✅ good */
--base-size: 20rem; /* ❌ bad */
```

The aspect ratio of the phone can also be modified using a custom property. The **CSS aspect-ratio property MUST NOT BE USED** as it will break the scale behavior. Instead, a custom property should be used instead.

```css
--aspect-ratio: 1 / 2.1; /* this is the default */

--aspect-ratio: 0.5; /* ✅ good */
aspect-ratio: 0.5; /* ❌ bad */
```

The phone preview container also accepts the following attributes/properties:

```typescript
type PhonePreviewProps = {
  locale?: string;
  clock?: string; // JSON-serialized Intl.DateTimeFormatOptions
};
```

**Clock Configuration:**
The `clock` property should be a JSON-serialized `Intl.DateTimeFormatOptions` object when used as an attribute:

```jsx
<sinch-labs-phone-preview
  locale="en-US"
  clock={JSON.stringify({ hour: '2-digit', minute: '2-digit', hour12: true })}
>
  <!-- content -->
</sinch-labs-phone-preview>
```

## RCS channel preview with custom content

The RCS channel preview component uses primitive props and slotted children for composition:

**Primitive Props:**
- `name` - Brand name (string)
- `description` - Brand description (string)
- `color` - Brand color (string)
- `banner` - Banner image URL (string)
- `logo` - Logo image URL (string)

**Slotted Content:**
Use `sinch-labs-phone-preview-rcs-channel-info-option` components as children with `slot="contacts"` to define contact options:

**Contact Option Props:**
- `type` - Contact type: "phone" | "website" | "email" (string)
- `label` - Display label (string)
- `contact` - Contact information (phone number, URL, or email) (string)
- `placeholder` - Whether this is a placeholder option (boolean)

```jsx
<sinch-labs-phone-preview-rcs-channel
  name="Brand Name"
  description="Brand Description"
  color="#007bff"
  banner="/banner.jpg"
  logo="/logo.png"
>
  <sinch-labs-phone-preview-rcs-channel-info-option
    slot="contacts"
    type="phone"
    label="Customer Service"
    contact="+1-555-0123"
  />
  <sinch-labs-phone-preview-rcs-channel-info-option
    slot="contacts"
    type="website"
    label="Visit Website"
    contact="https://example.com"
  />
  <sinch-labs-phone-preview-rcs-channel-info-option
    slot="contacts"
    type="email"
    label="Support Email"
    contact="support@example.com"
  />
</sinch-labs-phone-preview-rcs-channel>
```

## RCS chat preview

RCS chat design to be used in combination with the phone preview. Uses primitive props and slotted children for composition.

**Primitive Props:**
- `name` - Brand name (string)
- `description` - Brand description (string)
- `logo` - Brand logo image URL (string)

**Slotted Content:**
Use `sinch-labs-phone-preview-rcs-chat-message` components as children with `slot="messages"` to define chat messages:

**Message Props:**
- `text` - Message text content (string)

```jsx
<sinch-labs-phone-preview-rcs-chat
  name="Brand Name"
  description="Brand Description"
  logo="/logo.png"
>
  <sinch-labs-phone-preview-rcs-chat-message
    slot="messages"
    text="Hello! How can I help you today?"
  />
  <sinch-labs-phone-preview-rcs-chat-message
    slot="messages"
    text="We're here to assist you with any questions."
  />
</sinch-labs-phone-preview-rcs-chat>
```
