## Color Tokens

Our colors are stored as token values in our design system. Our tokens allow that color values are consistently used across our design system and the products that use it. They are divided into reference, system and component color tokens. In this section we will talk about the usage of reference and system colors and their correct usage.
Colors from component tokens are only used to build the component and nothing else.

<TokenIllustration />

### Reference colors

The reference colors stores all the raw values of all color allowed in our system.
This builds the basis of colors that are meant to be used by our system colors.

<ReferenceColorsTable />

### System colors

System colors  are used throughout the system in various contexts. They are essential for maintaining visual consistency and cohesiveness across the system. System colors are a critical when creating new themes for a design system. Themes allow designers to quickly and easily change the look and feel of an application, without having to manually update each individual UI element.

<br /> All values from the system colors comes from referenced values from the reference colors.
